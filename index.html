重新理解线段树及线段树的应用

发表于 2024-08-02

线段树是 OI 中的一种常见的数据结构，具有 O(nlog⁡2n)O\left(n\log_2n\right)O(nlog2​n) 的区间查询和修改复杂度，适用于维护满足结合律的区间信息。常见的线段树实现往往对于初学者来讲较难理解或容易出 Bug 但找不到，无从下手。这篇博客将介绍一种略有不同的线段树理解形式，有利于理解和实际应用并简化了问题。

前置结论

由线段树的定义方式，不难看出，线段树的每一个节点（表示一个连续区间）都具有高度的相似性，虽然底层的区间可能看似有所不同，但是实际上也能视为一个特殊的长度仅为 1 的区间，并包含与所有区间维护的同等的信息。

线段树实现区间快速查找依赖相邻两个区间信息可以合并为一个较大区间信息的性质。由此，我们得到了线段树的第一个基本操作：区间合并。

线段树的区间修改，我们在平时一种常见的解决方案是针对每一种修改做特殊化的处理，这里提出一个新的概念：修改信息（或增量信息） 用于存储一段区间的变化情况。

有了这个信息，就很容易自然的想到另外两个基本操作：修改信息的合并和修改信息应用到区间。

将三种基本操作合到一起看：

  1. 区间合并
  2. 修改信息的合并（可以理解为一个新的修改信息等效于依次进行两个修改的结果或者群论中的乘法）
  3. 修改信息应用到区间

思考

以上的三种操作是否可以用于解决相对复杂的区间问题呢？

如果我们这样想：

  • 区间信息不一定是一个单一的值，有可能是一个包含能够向上推导的信息的集合（学习过矩阵加速的读者可以联想，其实两者的思维方式非常类似）
  • 增量信息同样也可以包含多种修改

那么问题就被简化了，我们不再需要为了具体问题而纠结用算法维护的具体方式，所有的这些问题都可以用同一种操作逻辑来解决。

来看两个例子

例子1

写一棵线段树，要求实现以下三种操作：

  1. 区间求和
  2. 区间内所有数加上 x
  3. 区间内所有书乘上 x

需要完成区间加法和乘法，并维护区间求和

为了解决这个问题，需要维护一个 SUM 值（即区间求和信息），而增量信息包含 ADD ，MUL 两个子信息。

由于我们抽象了增量信息这个概念，因此加法和乘法都是同一种区间操作，只是对应的 (ADD⁡,MUL⁡)\left(\operatorname{ADD}, \operatorname{MUL}\right)(ADD,MUL) 值不同——加法为 (x,1)\left(x,1\right)(x,1) ，乘法为 (0,x)\left(0,x\right)(0,x)。

例子2

写一棵线段树，实现以下两种操作

  1. 将区间内所有数开平方根
  2. 求区间内所有数的和

思路是如果一段区间完全由 1 或 0 构成，则开平方根的新区间不会改变。

总结

线段树可以视为一种数据框架，面向不同的需求可以自定义其维护对象和对象之间的行为（即上文提到的三个基本操作），这样可以将线段树抽象为一种数学模型。
